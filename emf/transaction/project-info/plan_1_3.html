Introduction

This document lays out the feature and API set for the fourth release of the EMF Model Transaction (EMF MT) Project, version 1.3.0. This project plan inherits from the Modeling Project Plan, which should be referenced when consulting this individual project plan.

Release Deliverables

The release deliverables have the same form as is found in most Eclipse projects, namely: EMF MT source code release, available as versions tagged "R1_3" in the project's CVS repository. EMF MT SDK (includes runtime, sources, examples, and documentation) (downloadable and update site). EMF MT runtime binary distribution (downloadable and update site). EMF MT tests (downloadable)

Target Environments

In order to remain current, each Eclipse release targets reasonably current versions of the underlying operating environments. The EMF Model Transaction (EMF MT) project depends upon on the Platform and other projects, which are mostly "pure" Javaâ„¢. The 3.5 release of the Eclipse Platform Project is written and compiled against version 1.4 of the Java Platform APIs, and targeted to run on version 1.4 of the Java Runtime Environment, Standard Edition. EMF MT will target the same Java version as EMF Core, which currently requires Java 5. Eclipse Platform SDK 3.5 will be tested and validated on a number of reference platforms. EMF MT will be tested and validated against a subset of those listed for the platform.

Internationalization

The Eclipse Platform is designed as the basis for internationalized products. The user interface elements provided by the Eclipse SDK components, including dialogs and error messages, are externalized. The English strings are provided as the default resource bundles. As a result, the EMF MT project will provide English strings in its default bundles and be localized to a subset of those locales offered by the Platform. This plan will be updated to indicate which locales will be provided and the time frame for availability.

Themes and Priorities

Power to the People

Some relatively minor improvements in the API will give clients considerably more control over the behaviour and performance of their transactions.
  • Committed
    • Allow transaction reuse in direct command execution [245393] (target milestone: M2)
    • Allow clients to set options on existing instances of unexecuted AbstractEMFOperations [245419] (target milestone: M2)
    • Resource set listeners should be informed when they are added or removed from an editing domain [177642] (target milestone: M2)
    • Allow creation of custom NotificationFilter [225711] (target milestone: M2)
  • Proposed
    • Need pre / post hooks for Transaction execution [145877] (target milestone: 1.3)
    • ResourceUndoContext should be more flexible regarding affected resource policy [240775] (target milestone: 1.3)
    • Non-inherited transaction options prevent transaction reuse [245446] (target milestone: 1.3)
  • Deferred
    • No items.

Appealing to a Broader Community

The EMF MT project has a proven record of robustness and flexibility. However, some classes of application have not been able to realize the framework's full potential. These include highly dynamic applications that need to change meta-data at run-time, small-footprint RCP applications, and those that do not run in an instance of the Eclipse Platform.
