Henshin - Bank Account Example

This example shows some basic concepts of the Henshin transformation language and tool environment. The example transformation is endogenous, meaning that it operates on a single metamodel. We explain how to define transformation rules using Henshin's integrated graphical editor, and how to use the Interpreter Wizard to execute rules on an example model.

The example files can be downloaded here: bank.zip. You can import the example project directly into your Eclipse workspace.

Metamodel and Instance Model

The metamodel is defined in the file bank.ecore. We used the Ecore Tools to draw a diagram (depicted on the right). The diagram is stored in the file bank.ecorediag.

The class Bank serves as a root container for all other classes. Every bank owns a number of accounts, each with a unique ID and the current credit Two types of persons are distinguished: managers and clients. Every client is associated with at most one manager and can own a number of accounts in the bank.

An example instance model is depicted on the right using the Sample Reflective Ecore Model editor. It contains one manager, three clients and four accounts. Note that we did not generate code from the metamodel. Thus, we use dynamic EMF here, i.e. the instance model is typed over a dynamic metamodel loaded from bank.ecore.
